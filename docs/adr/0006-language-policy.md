# 0006 — Языковая политика: разделение по аудитории

- **Статус:** accepted
- **Дата:** 2026-06-29

## Контекст

Репозиторий выглядел противоречиво: `README.md`, commit-сообщения и идентификаторы
кода — на английском, а inline-комментарии, докстринги и вся `docs/` — на русском.
Со стороны это читается как недосмотр.

Аудит показал, что разнобоя на самом деле нет — есть последовательное разделение
по аудитории, просто оно нигде не было записано. Нужно решить: унифицировать всё
в один язык или закрепить разделение явно.

## Решение

Закрепляем **разделение по аудитории**, не унифицируем. Правило выбора:
«На кого смотрит текст?»

- **Английский** — внешняя витрина и код: идентификаторы кода, commit-сообщения,
  корневой `README.md`, технические маркеры-скаффолд в шаблонах (`PAGE HEAD`),
  литеральные имена внешних API-полей (напр. `Access token`/`Refresh token` у
  `AmoCRMAuth` — зеркалят OAuth amoCRM).
- **Русский** — всё для работающей RU-команды и заказчика: inline-комментарии,
  докстринги (формат Google-style, текст русский), `docs/` (Diátaxis + ADR),
  `CLAUDE.md`/`DEPLOY.md`/`OPERATIONS.md`, и всё пользовательское в админке
  (`verbose_name`, `help_text`, fieldsets).

Полная формулировка — `CLAUDE.md` §18.

## Последствия

**Плюсы:**

- `README.md` остаётся англоязычным showcase-артефактом для внешнего читателя.
- Рабочая документация и админка — на языке тех, кто реально их использует.
- Английские commit-сообщения (отраслевая конвенция) сохраняются.
- Правило записано (этот ADR + §18 + заметка в README) — новый код и доки
  следуют ему единообразно, разнобой не возникает заново.

**Минусы / плата:**

- Случайный читатель видит английский README рядом с русскими доками. Но теперь
  это объяснённое, зафиксированное решение, а не разнобой — заметка в README
  сразу отсылает к этому ADR.
